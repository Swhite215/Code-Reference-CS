        - Advantages of Ethernet
            - Security, Speed, Reliability, Cost, Compatibility
        - Ethernet Architecture
            - Topologies
                - Bus
                    - All devices are connected to a single bus, and each device analyzes every packet sent through the network
                    - Advantages - ease of use, low cost
                    - Disadvantages - high collision probability and degradation of performance as devices increase
                - Star
                    - All devices are connected to a central device
                    - Hub - Logical Bus - when a message arrives at the hub, it is broadcast to all connected devices
                    - Switch - Logical Bus - when a message arrives at the switch, it is sent directly to the destination device
            - Cable Types
                - Copper Coaxial, Copper Unshielded Twisted Pair Cabling, Fiber Optic Cabling
            - Carrier Sense Multiple Access/Collision Detection
                - Carrier Sense - listen to medium to determine if anyone is sending
                - Collision - two messages are sent at the same time and the signals collide
                - Jam Signal - emitted to indicate to all devices on the bus that sending will occur
                - Back Off - waiting a period of time before restarting the CSMA process
                - CSMA is not required with Ethernet and Switches
            - Ethernet Packet
                - Preamble, MAC Destination, MAC Source, Payload, Cyclic Redundancy Check Field

            - Provides host to host communication services between for applications
            - Services
                - Connection Establishment
                - Reliable/Unreliable Delivery
                - Flow Control
                - Congestion Control
                - Application Multiplexing
            - Protocols
                - Transmission Control Protocol
                    - TCP is a connection oriented, reliable protocol, that implements flow control, congrestion control, and ensures data integrity
                    - Application: situations where you need a guarantee the data will be delivered, in order, and unmodified e.g. HTTP, SMTP, SSH
                - User Datagram Protocol
                    - UDP is a connectionless, unreliable protocol, that does NOT implement flow or congestion control
                    - Application: situations where the application can tolerate loss in packets, and benefits from fast transmission of data e.g. VoIP, DNS, Online Gaming, ICM
            - Security
                - Secure Socket Layer - encryption, data integrity, and authentication mechanisms
                - Transport Layer Security - successor to SSL, HTTPS, SMTPS, POP3S, IMAPS, VoIP
                - Network Access Control/Access Control List - allow or deny access to a computer environment based on custom rules
                - Authentication and Authorization

                - Logical Link Control - flow control and transmission for analog and/or digital streams
                - Media Access Control Layer - protocol access to the physical medium, identificaiton, and authentication

                - Frame - unit of transmission, discernible block of information i.e. stream of bits
                - Problems
                    - Frame start/end, errors, overhead, compatibility
            - Error Control
                - Error Detection - has an error occured
                    - Causes - interference, manipulation
                    - Errors - Single Bit, Multiple Bits, Burst
                    - Techniques - Parity Check, Checksum, Cyclic Redundancy Check
                - Error Correction - fixing exact number of corrupted bits
                    - Forward Error Correction
                    - Backward Error Correction
        - Access Control Lists - filtering rules to identify allowed traffic
        - Firewalls
            - What - software or hardware based network security system to allow or deny traffic based on rules
            - Category
                - Network - operates at edge of private network
                - Host - opoeratees on a single computer in the network
            - Types
                - Packet Filtering - examining of IP packet header
                    - Evaluate source, destination, port, and service
                - Circuit - examining virtual circuit or sessions
                    - Evaluate session state
                - Applicaiton Layer - examining packet data portion
                    - Examine entirety of content, filter by user, group, and data
                    - Example - Proxy Server
                        - Forward Proxy - handles requests from inside a private network
                        - Reverse Proxy - handles request from the internet to a server located inside a private network
                - Unified Thread Management Device
                    - Features - firewall, VPN, anti-spam, anti-virus, load balancing
                - Next Generation Firewall
                    - Traditional firewall + other network device filtering functionalities
            - Rules
                - Implicity Deny - deny all traffic by default
                - Explicit Deny - deny permission to a specific user
        - Intrusion Detection System
            - Monitor and detect security breached, does NOT prevent an attack, but rather produces alerts
            - Detection Methods
                - Signature - packets unique fingerprint, known in database/dictionary, based on patterns
                - Anomaly - uses a defined baseline to identify traffic that falls out of it
            - Process - collect data -> analyzer -> generate alerts and log -> update dictionary
            - Traffic Assessment Types
                - Positive - attack detected
                - False-Positive - system misidentified harmless traffic as offensive
                - Negative - traffic harmless
                - False-Negative - system misidentified offensive traffic as harmless
            - Implementations
                - Host Based Intrusion Detection System - monitor system on single host
                - Network Based Intrusion Detection System - monitor traffic at edge of a network
                - Perimeter Intrusion Detection System - physical device
                - Virtual Machine Intrusion Detection System
        - Intrusion Prevention System
            - IDS + perform tasks to stop intrusion including
